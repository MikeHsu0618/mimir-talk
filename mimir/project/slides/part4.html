TSDB 的結構性不對

S3 天生適合大塊順讀。TSDB 為本地 SSD 的 random read 設計——搬到 S3 變成大量小 random GET。

Store Gateway 的連鎖代價

被迫 stateful · 昂貴本地 disk · sync 慢 · disk 壞 = 查詢掛。Cortex、Thanos、Mimir 三家共同的痛。

Request 數量才是成本，不是 bytes 數量。Parquet + 無狀態 querier → Store Gateway 的四項代價一次解掉。
